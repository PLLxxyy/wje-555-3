## 项目主要需求

### 项目概述

构建一个房产租赁管理平台（RentHub），为房东和租客提供房源发布、租约管理、账单收缴、维修工单等全流程租赁管理能力。平台支持三种角色（Landlord/Tenant/Admin），覆盖从房源上架到租约终止的完整租赁生命周期。

**项目英文短名（COMPOSE_PROJECT_NAME）**：`wjerenthub`

**技术栈**：Vue 3 + TypeScript + Element Plus（前端） / Django + Django REST Framework（后端） / MySQL 8.0 / Vite / JWT

**前端端口**：38205，**后端端口**：38305

**关系**：每条租约关联一个房源、一个租客、一个房东。一个房源同一时间只能有一个生效租约。

**租约状态流转**：
```
房东创建 → 待签 → 双方签约 → 生效
→ 距到期30天内 → 即将到期 → 到期日 → 已到期
→ 续签：创建新租约（状态为待签），旧租约标记为已终止
→ 提前终止：任意一方发起 → 已终止
```

**业务动作**：创建租约、签约确认、续签、终止（需填写原因）。

### 横切关注点

#### 1. 认证授权 + RBAC

- **后端**：
  - JWT 认证：自定义 JWT Authentication，Token 通过 Authorization Header 传递
  - 权限类（Permission Classes）：`IsLandlord`、`IsTenant`、`IsAdmin`、`IsOwner`
  - 数据范围过滤：Landlord 只能管理自己的房源和相关租约，Tenant 只能查看自己的租约和账单
  - Django Admin 后台：为 Admin 角色提供管理入口
- **前端**：
  - Vue Router 路由守卫：meta.requiresAuth、meta.roles 控制访问
  - 按钮级权限：v-permission 自定义指令控制按钮可见性
  - Axios 拦截器：请求头自动注入 JWT Token，响应 401 时清除 Token 跳转登录
  - 登录/注册页面：支持角色选择（房东/租客注册）
- **触达层**：DB → Django Model → Serializer → View → Permission → 前端路由守卫 → 按钮指令 → 数据范围

#### 2. 操作日志（审计）

- 通过 Django Signal 或自定义 Mixin 记录关键写操作
- 需要审计的操作：租约创建/签约/终止、账单创建/支付/减免、房源价格修改、维修工单状态变更
- 日志字段：操作人ID、操作类型、目标实体ID、旧值、新值、操作时间、IP 地址
- 审计日志表 AuditLog 存储于 MySQL
- 租约详情页面展示关联的操作日志时间轴

#### 3. 全局异常处理

- **后端**：Django REST Framework 自定义异常处理器（EXCEPTION_HANDLER），统一捕获业务异常和系统异常，返回标准错误响应 `{ code, message, details }`
- **前端**：Axios 响应拦截器，统一处理 4xx/5xx 错误，ElMessage 错误提示，401 自动跳转登录

## 代码存放规则

这是前后端分离项目。

前端代码放在 `frontend/`：
页面，组件，路由，状态管理，API 请求，类型定义，样式和资源等放在对应的子文件夹

后端代码放在 `backend/`：
数据库脚本放在 `database/`。

## 部署要求

项目必须支持 Docker Compose 一键部署，且在 **任意目录名（包括中文目录名）** 下都能正常启动。

需要提供以下文件：
- 根目录 `docker-compose.yml`：编排前端、后端、数据库及所有依赖服务
- 根目录 `.env`：包含 `COMPOSE_PROJECT_NAME=wjerenthub`，确保 Compose 项目名不依赖目录名
- `frontend/Dockerfile`：前端项目构建与运行镜像（多阶段构建，先构建再用 Nginx 托管）
- `backend/Dockerfile`：后端项目构建与运行镜像
- `frontend/nginx.conf`：Nginx 配置，处理前端路由和 API 反向代理
- `.env.example`：所有环境变量的示例文件，附注释说明；同时包含 `COMPOSE_PROJECT_NAME` 默认值

docker-compose.yml 要求：
- **不要写 `version:` 字段**
- 顶层声明 `name: wjerenthub`
- 所有服务使用 `container_name` 命名容器，带 `${COMPOSE_PROJECT_NAME:-wjerenthub}` 前缀
- 数据库数据使用命名卷持久化
- 前端通过 Nginx 反向代理后端 API（调用 `/api`，由 Nginx 转发到 `backend:38305`）
- 服务间通过 Docker 内部网络通信
- 数据库服务配置 `healthcheck`，后端 `depends_on: { db: { condition: service_healthy } }`
- 后端也配置 `healthcheck`，前端 `depends_on` 后端健康
- 暴露端口：前端 `38205:80`，后端 `38305:38305`
- 所有可变配置通过环境变量注入
